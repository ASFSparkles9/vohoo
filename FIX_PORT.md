# Исправление ошибки "Port already in use"

## Проблема:
```
Error: listen EADDRINUSE: address already in use :::3001
```

Это означает, что порт 3001 уже занят другим процессом.

## Решение 1: Остановить процесс (автоматически)

Процесс уже остановлен. Теперь запустите бэкенд снова:

```bash
npm run dev
```

## Решение 2: Остановить процесс вручную

### Windows PowerShell:
```powershell
# Найти процесс
netstat -ano | findstr :3001

# Остановить процесс (замените PID на номер из вывода выше)
Stop-Process -Id PID -Force
```

### Windows CMD:
```cmd
# Найти процесс
netstat -ano | findstr :3001

# Остановить процесс (замените PID на номер из вывода выше)
taskkill /PID PID /F
```

## Решение 3: Использовать другой порт

Если нужно использовать другой порт:

1. Измените `package.json`:
   ```json
   "dev": "next dev -p 3002"
   ```

2. Обновите `.env` файлы:
   - Корень: `NEXT_PUBLIC_APP_URL="http://localhost:3002"`
   - `mobile/.env`: `EXPO_PUBLIC_API_URL=http://192.168.3.3:3002`

3. Перезапустите бэкенд:
   ```bash
   npm run dev
   ```

## Проверка:

После остановки процесса, порт должен быть свободен. Проверьте:

```powershell
netstat -ano | findstr :3001
```

Если ничего не выводится - порт свободен.

## Предотвращение проблемы:

Всегда останавливайте бэкенд правильно:
- Нажмите `Ctrl+C` в терминале
- Не закрывайте терминал без остановки процесса
