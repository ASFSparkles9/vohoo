# Изменение порта приложения

## ✅ Порт изменен на 3001

Порт бэкенда изменен с **3000** на **3001**.

## Что было изменено:

1. ✅ `package.json` - добавлен флаг `-p 3001` для Next.js
2. ✅ `mobile/lib/api.ts` - обновлен дефолтный порт на 3001
3. ✅ `.env` в корне проекта - обновлен `NEXT_PUBLIC_APP_URL`

## Что нужно сделать вручную:

### 1. Обновите `.env` в папке `mobile`:

Откройте файл `mobile/.env` и измените:

**Было:**
```
EXPO_PUBLIC_API_URL=http://192.168.3.3:3000
```

**Должно быть:**
```
EXPO_PUBLIC_API_URL=http://192.168.3.3:3001
```

### 2. Перезапустите серверы:

```bash
# Остановите текущий бэкенд (Ctrl+C)
# Затем запустите заново:
npm run dev

# Бэкенд теперь будет на http://localhost:3001
```

### 3. Перезапустите мобильное приложение:

```bash
cd mobile
npm start
```

## Проверка:

1. Бэкенд должен быть доступен на: `http://localhost:3001`
2. Мобильное приложение должно подключаться к `http://192.168.3.3:3001`

## Если хотите использовать другой порт:

Измените порт в следующих местах:

1. `package.json` - строка `"dev": "next dev -p ВАШ_ПОРТ"`
2. `.env` (корень) - `NEXT_PUBLIC_APP_URL=http://localhost:ВАШ_ПОРТ`
3. `mobile/.env` - `EXPO_PUBLIC_API_URL=http://ВАШ_IP:ВАШ_ПОРТ`
4. `mobile/lib/api.ts` - дефолтное значение порта
