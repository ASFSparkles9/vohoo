# Инструкция по установке SyncNote

## Требования

- Node.js 18+ 
- npm или yarn

## Установка

1. **Установите зависимости:**
```bash
npm install
```

2. **Настройте переменные окружения:**
```bash
cp .env.example .env
```

Откройте `.env` и добавьте ваш OpenAI API ключ (опционально, для ИИ-функций):
```
OPENAI_API_KEY=your_key_here
```

3. **Инициализируйте базу данных:**
```bash
npx prisma generate
npx prisma db push
```

4. **Запустите приложение:**
```bash
npm run dev
```

Приложение будет доступно по адресу [http://localhost:3000](http://localhost:3000)

## Первый запуск

При первом запуске приложение автоматически создаст демо-пользователя. Все данные будут храниться локально в файле `prisma/dev.db`.

## Функции

### ✅ Реализовано

- 📝 Личные заметки с тегами и папками
- 📅 Умный календарь (день/неделя/месяц)
- ✅ Управление задачами
- 👥 Стыковка с друзьями
- 🤖 ИИ-помощник (требует OpenAI API ключ)
- 📚 История жизни
- ⚖️ Баланс жизненных сфер

### 🔄 В разработке

- Голосовой ввод (требует поддержку браузера)
- Drag & Drop для событий календаря
- Антипрокрастинационный режим
- Стыковка по геолокации
- Совместные заметки с ИИ-резюме

## Структура проекта

```
syncnote/
├── app/              # Next.js App Router
│   ├── api/          # API маршруты
│   └── page.tsx      # Главная страница
├── components/       # React компоненты
│   ├── notes/       # Компоненты заметок
│   ├── calendar/    # Компоненты календаря
│   ├── tasks/       # Компоненты задач
│   ├── friends/     # Компоненты друзей
│   ├── ai/          # ИИ-компоненты
│   └── life/        # История и баланс
├── lib/             # Утилиты
├── prisma/          # Схема базы данных
└── public/          # Статические файлы
```

## Разработка

### Просмотр базы данных:
```bash
npx prisma studio
```

### Обновление схемы базы данных:
```bash
npx prisma db push
```

## Примечания

- Для работы ИИ-функций требуется OpenAI API ключ
- Все данные хранятся локально в SQLite
- В продакшене рекомендуется использовать PostgreSQL
- Для аутентификации пользователей нужно добавить систему авторизации
