# Запуск бэкенда

## Важно: Бэкенд должен быть запущен!

Мобильное приложение не будет работать без запущенного бэкенда.

## Как запустить бэкенд:

### Шаг 1: Откройте новый терминал

**НЕ закрывайте терминал с Metro Bundler!** Откройте второй терминал.

### Шаг 2: Перейдите в корневую папку проекта

```bash
cd F:\vohoo
```

### Шаг 3: Запустите бэкенд

```bash
npm run dev
```

### Шаг 4: Дождитесь запуска

Вы должны увидеть:
```
▲ Next.js 14.0.4
- Local:        http://localhost:3001
- Ready in X ms
```

## Проверка работы бэкенда:

Откройте в браузере:
```
http://localhost:3001/api/init
```

Должен вернуться JSON:
```json
{"userId":"...","message":"Инициализация завершена"}
```

## Если бэкенд не запускается:

1. **Проверьте, что порт 3001 свободен:**
   ```powershell
   netstat -ano | findstr :3001
   ```
   Если порт занят, остановите процесс или измените порт.

2. **Проверьте базу данных:**
   ```bash
   npx prisma generate
   npx prisma db push
   ```

3. **Проверьте .env файл:**
   Убедитесь, что файл `.env` существует в корне проекта с:
   ```
   DATABASE_URL="file:./dev.db"
   ```

## Два терминала должны быть открыты:

1. **Терминал 1:** Metro Bundler (мобильное приложение)
   ```bash
   cd mobile
   npm start
   ```

2. **Терминал 2:** Бэкенд (Next.js)
   ```bash
   cd F:\vohoo
   npm run dev
   ```

## Порты:

- **Бэкенд:** http://localhost:3001
- **Metro Bundler:** http://localhost:8081
- **Веб-версия приложения:** http://localhost:8081 (после нажатия `w`)
